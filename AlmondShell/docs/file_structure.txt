│
│  NOTE: The ECS, component, and engine systems are now consumed via `import`
│  rather than `#include`. Pull the consolidated surface from `import aengine;`
│  or target partitions such as `import aengine.engine_components;` and
│  `import aengine.platform;` instead of the legacy umbrella headers. Legacy
│  headers stay mirrored behind the module surface (for example
│  `aengine.hpp`, `acontext.hpp`, `aatlasmanager.hpp`, `aatlastexture.hpp`,
│  `acommandline.hpp`, and `aplatform.hpp` are imported inside their matching
│  modules) so downstream users can mix `import` and `#include` during
│  migration.
│
│  Module consumption quick-start
│  • Importers: `aengine` (aggregate), `almondengine`/`almondshell` (umbrella
│    aliases), `aengine.engine_components`, `aengine.platform`,
│    `aengine.commandqueue`, `aengine.systems`, `aengine.context`/`*.context.*`,
│    `aengine.gui.menu`, `aengine.input`, `aengine.cli`, direct module mirrors
│    for legacy headers like `import aatlas.manager;`, `import aatlas.texture;`,
│    `import acommandline;`, `import aentity.*`, and the updater partitions
│    (`aengine.updater`, `.updater.config`, `.updater.system`, `.updater.tools`).
│    Renderer-specific partitions include `aengine.opengl.*`, `aengine.sdl.*`,
│    and `aengine.context.raylib`/`aengine.context.software`.
│  • CMake flags: configure with C++23 plus `-DCMAKE_CXX_SCAN_FOR_MODULES=ON`
│    (or `-DCMAKE_EXPERIMENTAL_CXX_MODULE_DYNDEP=ON` on 3.27–3.28) and a
│    module-capable compiler (`/std:c++latest` on MSVC, `-fmodules-ts` on
│    clang/GCC). Presets and build scripts set these by default.
│  • Mirrored headers: aengine.hpp, aplatform.hpp, acontext*.hpp, awindowdata.hpp,
│    acommandline.hpp, aguimenu.hpp, aatlasmanager.hpp, aopenglcontext.hpp, and
│    the legacy aentity*/acomponent* headers are imported inside their matching
│    partitions to keep macros and types aligned for mixed builds.
